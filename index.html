Images are an important part of the web and included on almost every webpage. They can capture the users attention, instantly convey messages and how they are used can make or break a website. It is important to get them right. Which is why we decided to make our project about images.

Img-Tag

The <img> tag is a fundamental element used to embed images into a web page. This tag allows you to display visual content, such as photographs, illustrations, icons and more.

<img src="image.jpg" alt="Description of the image">

You can also use external links as the source for your images by specifying a URL in the src attribute:

<img src="https://example.com/image.jpg" alt="Description of the image">

Image Position

The position attribute in CSS is a tool for controlling the layout of elements, especially images. Whether adjusting their position relative to the normal flow, placing them precisely within a container, or fixing them during scrolling, this attribute is an essential part in building a website.

Image Fit

Including images in a website often requires consideration of how they adapt to varying screen sizes and aspect ratios. object-fit provides a tool for controlling how an image (or any element) behaves within its container, particularly when dealing with dynamically sized elements.

See examples about image fit.

Aspect Ratio

The aspect ratio attribute can be used to set a containers aspect ratio without specifying its height or width. With aspect ratio: 1 / 1; for example an image can be forced to 1:1 instead of its initial ratio. If either width or height is set, the other will be scaled accordingly. If both are - aspect ratio will be ignored. aspect ratio: auto Is the default option - the element behaving as it usually would. When providing both auto and a ratio (e.g. aspect ratio: auto 2 / 1;) the ratio is used for normal elements and auto is applied to replaced elements like <img> or <iframe>.

Flexbox

Flexbox is a powerful layout model that simplifies the arrangement of images within a webpage. Flexbox allows for precise control over the alignment, distribution, and order of images, optimizing their presentation on different screen sizes.

Web Performance

Images take a long time to download (in comparison to the rest of the website). So it's important to think about what and how many images to use.
There are different kinds of images to choose from, these are the most common:

  • JPG
    Ideal for photographs and images with gradients.
    Lossy compression, which may result in some loss of quality.
    Moderate to small file size, depending on the compression level.
  • PNG
    For images with transparency or a need for high-quality graphics.
    Lossless compression, maintains high quality. Generally larger file size than JPG.
  • GIF
    Best for simple graphics, icons, and small animations.
    Limited to 256 colors.
    Supports simple animations through multiple frames.
    Small file size, especially for simple graphics and animations
  • SVG
    Ideal for logos, icons, and illustrations. Scalable without loss of quality.
    Generally small file size for simple graphics, but can vary.
  • WebP
    Provides both lossy and lossless compression. Generally offers smaller file sizes.
    Widely supported in modern browsers.
    Typically smaller file size than JPG or PNG.
  • AVIF
    A modern image format designed for high-quality compression, suitable for a wide range of images.
    Utilizes the AV1 video codec for efficient compression.
    Becoming increasingly supported in modern browsers.
    Generally offers excellent compression, resulting in smaller file sizes compared to JPEG and PNG while maintaining high quality.

Another important aspect is, when an image is loaded. The loading type is normally on eager (loading="eager").
That means that the image is loaded as soon as the webpage is loaded.
You can set loading="lazy" to only load the image if it is on-screen.
That can help, for example if you have a huge gallery. In the eager state, all images would be loaded at the start.

Accessibility

Because images are an important part of the web and included on basically every website, it's vital to ensure they are accessible to all users. This includes providing descriptive alt text for screen readers. Hover over the image to look at its alternative text (this can be done using the title-attribute).
